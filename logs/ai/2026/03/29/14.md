- AI 로그는 사람이 읽기 쉬운 Markdown 형식으로 남긴다.

## 실제 반영한 변경 사항

- `doc/ai 프롬포트 목록` 디렉터리를 `doc/ai` 구조로 재정리했다.
- 프롬프트 문서를 `doc/ai/prompts/base-agent.md`와 `doc/ai/prompts/code-review-agent.md`로 이동했다.
- `doc/ai/README.md`를 추가해 폴더 목적, 문서 목록, 관리 규칙을 정리했다.
- 예시 AI 로그 파일로 `logs/ai/2026/03/29/14.md`를 생성했다.

## 변경 결과 및 기대 효과

- 프롬프트 문서가 Markdown 파일로 통일되어 관리성이 좋아졌다.
- 이후 AI 작업 시 동일한 형식으로 로그를 누적할 수 있는 기준 파일이 생겼다.

## 추가 작업 일시

- 날짜: 2026-03-29
- 시간: 11:55

## 추가 사용자 요청 요약

- 베이스 에이전트 문서를 참고해 코드리뷰 에이전트 프롬프트를 읽고 수정해 달라는 요청
- 코드 리뷰 범위를 `git` 변경점 기준으로 잡아, 변경된 부분만 중심으로 리뷰하도록 `md`에 반영해 달라는 요청

## 추가 사전 합의한 핵심 내용

- 수정 대상은 `doc/ai/prompts/code-review-agent.md` 문서다.
- 코드 리뷰는 전체 코드베이스 재검토가 아니라 `git diff` 기반 변경 범위를 우선으로 삼아야 한다.
- 변경 사항은 기존 AI 로그 형식에 맞춰 같은 날짜 로그 파일에 추가 기록한다.

## 추가 실제 반영한 변경 사항

- `doc/ai/prompts/code-review-agent.md`에 리뷰 범위를 정의하는 섹션을 추가했다.
- 리뷰 시작 전 `git status`, `git diff --stat`, `git diff`로 변경 파일과 hunk를 식별하도록 명시했다.
- 변경 범위 밖의 코드는 현재 수정과 직접 연결되는 경우에만 제한적으로 검토하도록 규칙을 보강했다.
- `git` 기준 변경점이 없거나 비교 기준이 불명확하면 사용자에게 대상 브랜치나 커밋을 확인하도록 추가했다.

## 추가 변경 결과 및 기대 효과

- 코드리뷰 에이전트가 전체 저장소를 광범위하게 훑지 않고 실제 변경분 중심으로 더 정확하게 리뷰할 수 있게 됐다.
- 리뷰 결과가 현재 작업 범위와 더 직접적으로 연결되어 불필요한 지적을 줄일 수 있게 됐다.
