# AI 작업 로그

## 작업 일시

- 날짜: 2026-03-29
- 시간: 13:09

## 사용자 요청 요약

- 코드 리뷰 에이전트를 Markdown 리뷰 파일을 남기는 형식으로 바꾸고, 사용자가 체크한 항목만 반영되도록 수정해 달라는 요청
- 반영하지 않은 항목은 따로 모아서 볼 수 있도록 로그 방식도 함께 정리해 달라는 요청

## 사전 합의한 핵심 내용

- 코드 리뷰 결과는 채팅 응답뿐 아니라 별도 Markdown 파일로 남긴다.
- 리뷰 항목은 체크박스 기반으로 작성하고, 사용자가 체크한 항목만 실제 반영 대상으로 본다.
- 미반영 항목은 별도 누적 로그에서 다시 볼 수 있어야 한다.

## 실제 반영한 변경 사항

- `doc/ai/prompts/code-review-agent.md`에 코드 리뷰 결과를 `logs/code-review/YYYY/MM/DD/HH.md` 형식의 Markdown 파일로 기록하도록 규칙을 추가했다.
- 각 리뷰 항목에 체크박스와 고유 ID를 부여하고, 사용자가 체크한 항목만 반영 대상으로 삼도록 명시했다.
- 체크되지 않은 항목은 `logs/code-review/pending.md` 같은 별도 누적 로그로 관리하도록 규칙을 추가했다.
- 코드 리뷰 문서에 리뷰 파일 구조와 예시 Markdown 형식을 함께 추가했다.
- `logs/code-review/pending.md` 파일을 생성해 미반영 리뷰 항목 누적 기준 파일을 마련했다.

## 변경 결과 및 기대 효과

- 코드 리뷰 결과가 파일로 축적되어 이후 추적과 재검토가 쉬워졌다.
- 사용자가 선택한 항목만 반영하는 흐름이 명확해져 불필요한 수정 가능성이 줄어들었다.
- 미반영 항목을 따로 모아 관리할 수 있어 누락된 리뷰 이슈를 다시 확인하기 쉬워졌다.

## 사용자 요청 요약

- 모든 로그에 공통으로 적용할 수 있도록 제목과 내용이 분리되고 작업 끝에 구분선이 들어가는 형식으로 바꿔 달라는 요청

## 사전 합의한 핵심 내용

- 공통 로그 형식 규칙은 베이스 에이전트 문서에 명시한다.
- 코드 리뷰 로그도 같은 형식을 따르도록 코드리뷰 에이전트 문서에 반영한다.
- 같은 시간대의 AI 작업 로그는 기존 시간 파일에 이어 쓰되 작업 단위 끝에 구분선을 넣는다.

## 실제 반영한 변경 사항

- `doc/ai/prompts/base-agent.md`에 모든 로그를 제목과 내용 섹션으로 작성하고 작업 끝에 `---` 구분선을 넣도록 규칙을 추가했다.
- `doc/ai/prompts/code-review-agent.md`에 코드 리뷰 로그도 같은 형식을 따르도록 규칙을 추가했다.
- 코드 리뷰 로그 예시에 작업 종료 구분선 `---` 를 추가했다.

## 변경 결과 및 기대 효과

- 로그 파일에서 작업 단위 경계가 더 명확해져 사람이 읽고 추적하기 쉬워졌다.
- AI 로그와 코드 리뷰 로그가 같은 시각적 규칙을 따라 일관성이 높아졌다.
