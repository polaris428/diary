## 작업: 일기 수정 기능(MVP 5번) 구현
- **작업 날짜와 시간**: 2026-05-07 01:44 - 01:49
- **사용자의 요청 요약**: 일기 상세 화면 내에서 바로 제목, 내용, 감정을 수정할 수 있는 편집 기능 구현.
- **사용자와 사전 합의한 핵심 내용**: 
  - 명세서에는 없었으나 제목(`title`)도 수정할 수 있도록 포함.
  - 저장/취소 버튼은 명세서의 하단 배치가 아닌, 상단 헤더(`AppHeader` 우측 슬롯)에 배치하여 읽기 모드와 통일성 유지.
- **실제로 반영한 코드, 문서 변경 사항 요약**:
  - `features/diary-write/composables/useDiaryWrite.ts`: Supabase의 `UPDATE` 구문을 활용하는 `updateEntry(id, payload)` 메서드 추가 구현.
  - `pages/diary/[id].vue`: 
    - `isEditMode`, `editTitle`, `editContent`, `editMood`, `isSubmitting` 등 로컬 상태 선언.
    - 헤더 우측에 읽기 모드(수정/삭제)와 편집 모드(취소/저장) 버튼을 `v-if`로 동적 전환.
    - 편집 모드 시 제목 `<input>`, `readonly` 해제된 `<BaseTextarea>`, `<DiaryMoodPicker>` 렌더링.
    - `isSaveDisabled` computed: 제목 또는 본문이 비어있으면 저장 버튼 비활성화.
    - `saveEdit`: 저장 성공 시 `entry.value`를 응답 데이터로 즉시 갱신하여 화면 반영.
  - `doc/기능/1차 mvp/5.일기 수정.md`: 수용 기준(AC) 모두 완료(`[x]`) 처리.
- **변경 결과 또는 기대 효과**: 상세 보기 화면 안에서 수정 모드로 진입하여 제목·내용·감정을 수정하고 저장할 수 있으며, 성공 시 변경된 데이터가 화면에 즉시 반영됨.
