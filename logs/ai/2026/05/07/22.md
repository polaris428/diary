## 작업 개요
DB 설계 문서(`db설계.md`)와 실제 Supabase DB 및 프론트엔드 코드 간의 정합성을 검토하고, 불일치 사항을 수정함.

## 변경 사항

### 문서 수정 (`doc/기능/db 설계/db설계.md`)
- `entries` 테이블의 `title` 컬럼 제약을 `NULLABLE`에서 `NOT NULL`로 변경 (실제 DB 스펙에 맞춤)
- `updated_at` 자동 갱신 방식을 DB 트리거(`set_updated_at`) 사용으로 확정 기록

### 검토 내용
- **RLS 정책**: 현재 DB의 서술형 정책명을 관례적인 `entries_select_own` 형태로 변경하는 SQL 제안.
- **인덱스**: 중복되거나 불필요한 인덱스(`entries_user_id_idx`, `entries_created_at_idx`) 삭제 SQL 제안.
- **트랜잭션**: 중요 DB 작업 시 안전을 위한 트랜잭션 개념 설명.
