# 디자인 컨셉 및 시스템 2D 갤러리로 전환 (3D -> 2D)

## 작업 일시
- 날짜: 2026년 5월 14일
- 시간: 23시 28분

## 사용자 요청 요약
- 3D 공간 기반이었던 'The Frame' 갤러리 디자인 시스템을, 리소스 최적화 및 모바일 접근성 향상을 위해 2D 기반으로 수정 요청.
- 단, 추후 사용자가 액자 위치를 다르게 하거나 새로운 오브젝트를 배치할 수 있는 '커스텀 확장성(Custom Wall)'을 고려할 것.
- AI 모델: 문서 작업에 효율적인 Gemini 3 Flash 사용(작업 중간에 3.1 Pro로 모델 변경 인지함).
- 디자인/레이아웃 모델: 2D 살롱 스타일 기반의 '상대 좌표 캔버스 모델(Relative Coordinate Model)' 제안 및 채택.

## 사전 합의된 핵심 내용
- **2D 갤러리 뷰**: 3D 카메라 줌인 효과 대신, CSS Transform 기반의 Scale & Translate를 활용한 2D 줌/페이드 전환으로 UX(Dive) 개편.
- **상대 좌표 모델**: 향후 드래그 앤 드롭 등 커스텀 확장을 위해 캔버스를 기준으로 x, y(%) 값을 갖는 유연한 배치 규칙 도입.
- **문서 동기화**: `디자인 컨셉.md`와 `디자인 시스템.md`에서 TresJS, 3D 조명/메쉬 관련 내용을 제거하고 2D 레이어링/그림자/모션 요소로 치환.

## 변경 사항 요약
### 1. `doc/설계도/디자인 컨셉.md` 수정
- 비전 명시: 3D 뮤지엄 -> "프리미엄 2D 갤러리 월(Gallery Wall)"
- 3D 요소(바닥 반사광 등)를 2D CSS 레이어링, 그림자(Box-shadow) 등으로 대체.
- 배치 방식에 '상대 좌표 모델' 항목 추가 및 향후 커스텀 모드 확장 가능성 기술.
- 다이브(Dive) 경험을 2D 화면 줌(Scale & Translate) 방식으로 변경.

### 2. `doc/설계도/디자인 시스템.md` 수정
- 불필요한 토큰 삭제: `gallery.floor` 및 3D 관련 Lighting/Shadow.
- 새로운 토큰 추가: `Wall.Coordinate`, `Frame.OuterShadow`, `Frame.InnerShadow`.
- 컴포넌트 역할 재정의: `BaseButton`과 `BaseFrame`을 2D 환경 및 좌표 기반 확장에 맞게 설명 업데이트.

## 기대 효과
- 리소스가 많이 드는 3D 개발 일정을 제거함으로써 코어 UX와 모바일 최적화에 집중할 수 있게 됨.
- 향후 '커스텀 월(Custom Wall)' 요구사항이 생겼을 때, 구조를 뒤엎지 않고 즉각적으로 2D 좌표 기반 기능을 추가할 수 있는 아키텍처 토대 마련.
