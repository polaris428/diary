# 핵심 원칙

> 상세 규칙은 `doc/설계도/아키텍처.md`를 기준으로 한다.

## 1. 구조

- 기본 CRUD 경로는 `Page -> Store -> Supabase`
- Nuxt `server/api`는 기본 경로가 아니다
- 모듈 경계는 `app / features / shared / core`
- 단방향 데이터 흐름만 허용한다

## 2. 보안

- 인증과 데이터 보호의 1차 책임은 Supabase Auth + RLS
- 클라이언트 분기는 보조 수단이다

## 3. 책임 분리

- `app`: 라우트, 레이아웃, 미들웨어, 조합
- `features`: 기능별 UI, 상태, 액션, 타입
- `shared`: 공통 UI, 공통 유틸, 공통 타입
- `core`: auth, session, supabase, error 같은 전역 기반

## 4. 구현 규칙

- Store만 Supabase를 호출한다
- `pages`는 조합과 화면 분기만 담당한다
- Component는 Props/Emit으로만 통신한다
- shared는 진짜 공통일 때만 사용한다
- 구조 변경이 필요하면 구현보다 설계도 수정을 먼저 한다
