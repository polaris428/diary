# 디자인 시스템

> 토큰 정의 기준: `design-system/tokens.ts`

## 1. 원칙

- 색상값 직접 하드코딩 금지
- 공통 컴포넌트는 토큰만 소비한다
- 기능 정책은 디자인 시스템 문서에 쓰지 않는다
- 기능별 화면 흐름은 기능 문서에서 다룬다

## 2. 컬러 토큰

| 그룹 | 역할 | 특성 |
| --- | --- | --- |
| `primary` | 주요 액션·강조 | 베이지/브라운 계열 5단계 (`50 ~ 900`) |
| `neutral` | 텍스트·배경·구분선 | 회백색 계열 6단계 (`0 ~ 900`) |
| `semantic.error` | 에러·파괴적 액션 | 레드 계열 |
| `semantic.success` | 성공 피드백 | 그린 계열 |
| `overlay` | 모달 배경 | 반투명 블랙 |

## 3. 감정 토큰

| Mood | 한글 | 이모지 | 배경 계열 | 텍스트 계열 |

| `complex` | 복잡함 | 😐 | 연회색 | 다크그레이 |
| `tired` | 피곤함 | 😴 | 연보라 | 퍼플 |
| `anxious` | 불안함 | 😟 | 연레드 | 레드 |
| `sad` | 슬픔 | 😢 | 연블루 | 네이비 |

## 4. 타이포그래피 토큰

| 토큰 | 폰트 | 사용처 |
| --- | --- | --- |
| `font.serif` | Noto Serif KR | 일기 본문 |
| `font.sans` | Pretendard | UI 전반 |

| 사이즈 | px | 사용처 |
| --- | --- | --- |
| `text.xs` | 12 | 날짜, 글자 수 카운터 |
| `text.sm` | 14 | 카드 미리보기, 레이블 |
| `text.base` | 16 | 기본 본문 |
| `text.lg` | 18 | 소제목 |
| `text.xl` | 22 | 화면 제목 |
| `text.2xl` | 28 | 홈 CTA |

- 줄 간격: `1.4` / `1.7` / `2.0`

## 5. 간격 · 반경 · 이펙트 토큰

| 구분 | 토큰 | 값 |
| --- | --- | --- |
| 간격 | `spacing.1 ~ spacing.16` | 4px 기반 스케일 (`4px ~ 64px`) |
| 반경 | `radius.sm / md / lg / full` | `6px / 12px / 16px / 9999px` |
| 그림자 | `shadow.sm / md / lg` | 3단계 |
| 트랜지션 | `transition.fast / normal` | `120ms / 200ms` |

## 6. 공통 컴포넌트

공통 컴포넌트는 기본적으로 `shared/components`에 둔다.

### BaseButton

| 항목 | 내용 |
| --- | --- |
| 역할 | 서비스 내 모든 버튼의 기본 단위 |
| Props | `variant / size / disabled / loading / fullWidth` |
| Variants | `primary` / `ghost` / `danger` |
| Sizes | `sm / md / lg` |
| loading 상태 | 텍스트 대신 스피너 표시 + 클릭 이벤트 차단 |
| disabled 상태 | `opacity 0.4` • `cursor not-allowed` |

### BaseTextarea

| 항목 | 내용 |
| --- | --- |
| 역할 | 본문 입력 및 읽기 전용 표시 |
| Props | `modelValue / placeholder / maxLength(5000) / showCounter / readonly` |
| 폰트 | `font.serif` 적용 |
| 높이 | 최소 200px, 내용 따라 auto-grow |
| 카운터 | 현재 글자 수 / 5000 |

### BaseInput

| 항목 | 내용 |
| --- | --- |
| 역할 | 단일 텍스트 입력 |
| Props | `modelValue / placeholder / error` |
| 에러 상태 | 하단 에러 텍스트 + 에러 보더 |

### BaseDialog

| 항목 | 내용 |
| --- | --- |
| 역할 | 확인/취소가 필요한 모달 |
| Props | `open / title / confirmText / cancelText / variant / loading` |
| Emits | `confirm / cancel` |
| 오버레이 | 클릭 시 닫힘 |
| 버튼 배치 | 취소 → 확인 순서 |

### BaseToast

| 항목 | 내용 |
| --- | --- |
| 역할 | 성공/에러 피드백 알림 |
| Props | `message / type / duration` |
| 위치 | 화면 하단 중앙 고정 |
| 인스턴스 | 앱 전역 1개 |

### BaseBadge

| 항목 | 내용 |
| --- | --- |
| 역할 | 감정 뱃지 시각화 |
| Props | `mood` |
| 스타일 | mood 토큰 자동 적용 |

### BaseEmptyState

| 항목 | 내용 |
| --- | --- |
| 역할 | 데이터 없을 때 빈 상태 UI |
| Props | `message / actionText` |
| Emits | `action` |

### AppHeader

| 항목 | 내용 |
| --- | --- |
| 역할 | 상단 공통 헤더 |
| Props | `title / showBack` |
| 슬롯 | `#right` |

### AppBottomNav

| 항목 | 내용 |
| --- | --- |
| 역할 | 하단 탭 네비게이션 |
| 탭 구성 | 홈(`/home`) / 기록(`/diary`) / 설정(`/settings`) |
| 활성 탭 | 현재 라우트 기반 자동 감지 |
| 표시 조건 | 상위 조합 레이어가 제어 |
