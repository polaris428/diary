# 공통 로그 작성 규칙

AI 에이전트가 실제 코드, 문서, 리뷰 결과를 파일로 기록할 때는 아래 규칙을 공통으로 따른다.

## 목적

- 로그의 목적은 AI 작업 이력을 사람이 추적하고 디버깅 가능하게 만드는 것이다.
- 아직 사용자와 충분히 합의되지 않은 아이디어나 검토 중인 내용은 확정 로그처럼 남기지 않는다.

## 파일 경로와 이름

- 로그는 날짜별 디렉토리 또는 파일 체계로 관리하며, 연/월/일 기준으로 저장한다.
- 로그 파일명은 실제 작업이 수행된 로컬 시각의 시간을 기준으로 정한다.
- 서로 다른 시간대의 작업은 각각 해당 시간 파일에 기록하며, 기존 로그 파일이 있다는 이유만으로 다른 시간 파일에 이어 쓰지 않는다.
- 예시 형식:
  - `logs/ai/2026/03/29/14.md`
  - `logs/code-review/2026/03/29/13.md`
  - `logs/ai/2026-03-29/14-00.md`

## 작성 형식

- 로그는 반드시 사람이 읽기 쉬운 Markdown 또는 텍스트 형식으로 작성한다.
- 모든 로그는 제목과 그에 대응하는 내용을 섹션 단위로 분리해 작성한다.
- 각 작업 기록의 끝에는 반드시 Markdown 구분선 `---` 를 넣어 사람이 다음 작업 단위를 쉽게 구분할 수 있게 한다.
- 같은 파일에 여러 작업을 누적할 때도 각 작업 단위 끝마다 동일하게 `---` 를 넣는다.

## 필수 내용

- 작업 날짜와 시간
- 사용자의 요청 요약
- 사용자와 사전 합의한 핵심 내용
- 실제로 반영한 코드, 문서, 리뷰 변경 사항 요약
- 변경 결과 또는 기대 효과
