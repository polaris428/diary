## 18. 이벤트 트래킹

> MVP에서는 외부 analytics 없이 Supabase 테이블 직접 집계로 측정한다.
> 

### 측정 지표 3개

| 지표 | 정의 | 집계 방법 |
| --- | --- | --- |
| 첫날 작성 완료율 | 가입 당일 entry 1건 이상 작성 비율 | `entries.created_at::date = auth.users.created_at::date` |
| 7일 재방문율 | 가입 후 7일 내 2일 이상 작성 비율 | `distinct date count ≥ 2` |
| 작성 이탈율 | `/write` 진입 후 저장 없이 이탈 | `write_start` 대비 `entry_saved` 미발생 비율 |

### 클라이언트 이벤트 정의

| 이벤트명 | 발생 시점 |
| --- | --- |
| `write_start` | `/write` 페이지 마운트 |
| `entry_saved` | `createEntry` 성공 |
| `entry_deleted` | `deleteEntry` 성공 |
| `all_entries_deleted` | `deleteAllEntries` 성공 |
- Phase 2에서 Mixpanel 또는 PostHog 전환 검토
- MVP는 `console.log` 수준 유지
