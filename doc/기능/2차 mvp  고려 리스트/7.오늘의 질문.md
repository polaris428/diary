## 기능 07 — 오늘의 질문

### 개요

| 항목 | 내용 |
| --- | --- |
| 위치 | `utils/question.ts`, `composables/useQuestion.ts` |
| 실행 | 클라이언트 순수 계산 |
| 알고리즘 | `dayOfYear % 30` |
| 사용 화면 | `/write` |

### 유저 스토리

> 나는 작성 화면에서 오늘에 맞는 질문을 보고 글쓰기 막힘을 해소하고 싶다.
> 

### 알고리즘 명세

| 항목 | 내용 |
| --- | --- |
| 입력 | 오늘 날짜 (`Date`) |
| 연산 | `dayOfYear` 계산 → `% 30` |
| 출력 | 질문 문자열 1개 |
| 특성 | 같은 날 동일 질문 / 날짜 변경 시 변경 / 외부 의존 없음 |

## 기능 08 — 전체 데이터 삭제

### 개요

| 항목 | 내용 |
| --- | --- |
| 화면 | `pages/settings.vue` |
| Store | `diary.store.deleteAllEntries() / auth.store.signOut()` |
| Supabase | `DELETE WHERE user_id = auth.uid()` |
| 성공 후 | 설정 화면 유지 + 성공 토스트 |

### 유저 스토리

> 나는 내 모든 일기를 한 번에 영구 삭제하고 싶다.
> 

### 설정 화면 전체 구성

| 요소 | 내용 |
| --- | --- |
| 유저 정보 | 이름·이메일 표시 |
| BaseButton ghost | 로그아웃 |
| 구분선 | 섹션 분리 |
| BaseButton danger | 내 기록 전체 삭제 |
| 개인정보처리방침 | 외부 링크 |
| 서비스 버전 | 하단 표시 |

### 2단계 확인 플로우

1. 내 기록 전체 삭제 클릭 → `step = 1`
2. 1단계 다이얼로그: "모든 기록이 영구 삭제돼요." → 계속
3. 2단계 다이얼로그: `BaseInput` 에 `삭제` 입력 시만 최종 삭제 버튼 활성화
4. 최종 삭제 클릭 → `deleteAllEntries()`
5. 성공: 상태 초기화 + 성공 토스트
6. 실패: 에러 토스트 + 다이얼로그 유지

### 페이지 로컬 상태

| 상태 | 설명 |
| --- | --- |
| `step` | `0 / 1 / 2` |
| `confirmText` | `삭제` 입력 확인용 |

### 엣지케이스

| 상황 | 처리 |
| --- | --- |
| entries 0건 | 정상 완료 처리 |
| 다른 텍스트 입력 | 최종 삭제 버튼 disabled |
| 삭제 중 네트워크 단절 | 에러 토스트 + 다이얼로그 유지 |

### 수용 기준 (AC)

- [ ]  `삭제` 입력 없이는 최종 삭제 버튼이 활성화되지 않는다
- [ ]  취소는 어느 단계에서든 상태가 초기화된다
- [ ]  삭제 후 `/diary` 에서 빈 상태가 표시된다
- [ ]  RLS 에 의해 본인 데이터만 삭제된다
