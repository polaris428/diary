## 기능 04 — 일기 상세

### 개요

| 항목 | 내용 |
| --- | --- |
| 화면 | `pages/diary/[id].vue` 읽기 모드 |
| Store | `fetchEntry(id) / currentEntry` |
| Supabase | `SELECT single (id + user_id 조건)` |
| 404 처리 | id 없음 또는 타인 id 접근 |

### 유저 스토리

> 나는 특정 일기를 탭해 전체 내용과 감정을 읽고 싶다.
> 

### 화면 구성

| 컴포넌트 | Props / 역할 |
| --- | --- |
| AppHeader | `showBack=true`, 우측 수정·삭제 버튼 |
| 날짜 헤더 | 포맷 예: `2026년 2월 19일 목요일` |
| BaseBadge | mood 없으면 미표시 |
| BaseTextarea | `readonly=true` |
| BaseButton ghost | 수정 모드 진입 |
| BaseButton danger | 삭제 다이얼로그 노출 |

### 플로우

- `/diary/:id` 진입
- `fetchEntry(id)` 실행
- 없음: "기록을 찾을 수 없어요." + `/diary` 돌아가기 버튼
- 있음: 날짜, 감정, 본문 렌더링

### 수용 기준 (AC)

- [ ]  본인 entry 접근 시 전체 내용이 렌더링된다
- [ ]  타인 entry 직접 URL 접근 시 404 처리된다
- [ ]  mood가 null이면 BaseBadge가 표시되지 않는다
