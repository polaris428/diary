## 기능 02 — 일기 작성

### 개요

| 항목 | 내용 |
| --- | --- |
| 화면 | `pages/write.vue` |
| Store | `diary.store.createEntry()` |
| Supabase | `entries INSERT` |
| 성공 후 | BaseToast "기록됐어요." → `/home` |

### 유저 스토리

> 나는 본문을 쓰고 감정을 선택해 1분 안에 기록을 저장하고 싶다.
> 

### 화면 구성

| 컴포넌트 | Props / 역할 |
| --- | --- |
| AppHeader | `title="오늘의 기록"`, `showBack=true` |
| DiaryQuestionBanner | `question` — `useQuestion()` 계산 결과 |
| BaseTextarea | `v-model=content`, `maxLength=5000`, `showCounter=true` |
| DiaryMoodPicker | `v-model=selectedMood` |
| BaseButton primary | 저장하기, `disabled=content 비어있음`, `loading=저장 중` |

### 페이지 로컬 상태

| 상태 | 설명 |
| --- | --- |
| `content` | 본문 입력값 |
| `selectedMood` | 선택된 감정 |
| `isDirty` | 입력 시작 후 true |

### 플로우

1. `/write` 진입
2. `useQuestion()` 으로 오늘의 질문 계산
3. 사용자 입력 → `isDirty = true`
4. 저장 버튼 클릭 → `diary.store.createEntry({ content: content.trim(), mood: selectedMood })`
5. 성공: `isDirty = false` → BaseToast → `/home`
6. 실패: 에러 토스트 + 입력값 유지
7. 미저장 이탈 시: `beforeunload + onBeforeRouteLeave` confirm 노출

### 검증 규칙

| 규칙 | 처리 |
| --- | --- |
| content 비어있음 | 저장 버튼 disabled |
| 공백만 입력 | `trim()` 후 0자 → disabled |
| 5000자 초과 | maxLength + DB constraint 이중 방어 |
| mood 미선택 | `null` 저장 허용 |
| 저장 실패 | content 유지 |

### 수용 기준 (AC)

- [ ]  content 1자 이상일 때만 저장 버튼이 활성화된다
- [ ]  저장 성공 시 row 생성 후 `/home` 으로 이동한다
- [ ]  저장 실패 시 입력값이 사라지지 않는다
- [ ]  미저장 이탈 시 confirm 이 노출된다
- [ ]  mood 미선택 저장 시 DB에 null 로 저장된다
