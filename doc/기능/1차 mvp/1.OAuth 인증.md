## 기능 01 — 구글 OAuth 인증

### 개요

| 항목 | 내용 |
| --- | --- |
| 화면 | `pages/index.vue`, `pages/auth/callback.vue` |
| Store | `auth.store — signInWithGoogle() / initSession()` |
| Supabase | `supabase.auth.signInWithOAuth({ provider: 'google' })` |
| 성공 후 | `/home` 이동 |

### 유저 스토리

> 나는 구글 계정 선택 한 번으로 로그인하고 싶다.
> 

### 화면 구성 — `pages/index.vue`

| 컴포넌트 | 역할 |
| --- | --- |
| 서비스 로고 | 브랜드 진입점 |
| 소개 문구 | "오늘도, 기록할 준비가 됐나요." |
| BaseButton primary | "구글로 시작하기" + 구글 로고 아이콘 |
| 에러 문구 | OAuth 실패·취소 시 버튼 하단 인라인 표시 |

### 화면 구성 — `pages/auth/callback.vue`

| 요소 | 역할 |
| --- | --- |
| 로딩 UI | "로그인 중..." 표시 |
| 세션 확인 | 성공 → `/home`, 실패 → `/?error=oauth_error` |

### 플로우

- `/` → 구글로 시작하기 클릭
- OAuth 동의 화면 이동
- 취소 시: `/auth/callback?error=access_denied` → `/?error=access_denied`
- 성공 시: 세션 생성 → `/auth/callback` → `/home`

### 검증 규칙

| 상황 | 처리 |
| --- | --- |
| OAuth 취소 | "로그인이 취소됐어요." |
| OAuth 서버 오류 | "로그인에 실패했어요. 다시 시도해주세요." |
| 콜백 도달 시 세션 없음 | `/?error=oauth_error` 리다이렉트 |
| 세션 있는 상태로 `/` 접근 | `/home` 리다이렉트 |
| 세션 없이 보호 라우트 접근 | `/` 리다이렉트 |
