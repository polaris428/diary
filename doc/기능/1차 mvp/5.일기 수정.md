## 기능 05 — 일기 수정

### 개요

| 항목 | 내용 |
| --- | --- |
| 화면 | `pages/diary/[id].vue` 수정 모드 |
| Store | `diary.store.updateEntry(id, data)` |
| Supabase | `entries UPDATE` |
| 성공 후 | 읽기 모드 복귀 |

### 읽기 모드 ↔ 수정 모드 전환

| 요소 | 읽기 모드 | 수정 모드 |
| --- | --- | --- |
| BaseTextarea | `readonly=true` | `readonly=false` |
| DiaryMoodPicker | 미표시 | 표시 |
| 하단 버튼 | 수정하기 / 삭제 | 저장하기 / 취소 |

### 페이지 로컬 상태

| 상태 | 설명 |
| --- | --- |
| `isEditMode` | 수정 모드 여부 |
| `editContent` | 편집 중 본문 |
| `editMood` | 편집 중 감정 |

### 플로우

1. 수정하기 클릭
2. 원본값 복사
3. 저장하기 클릭 → `updateEntry(id, { content, mood })`
4. 성공: 읽기 모드 복귀 + 최신값 반영
5. 실패: 에러 토스트 + 편집값 유지
6. 취소: 원본 복원

### 검증 규칙

| 규칙 | 처리 |
| --- | --- |
| content 비어있음 | 저장 버튼 disabled |
| RLS | 타인 entry UPDATE 거부 |
| updated_at | 현재 시각으로 갱신 |

### 수용 기준 (AC)

- [ ]  수정 저장 후 변경된 내용이 즉시 반영된다
- [ ]  취소 시 원본 내용으로 복원된다
- [ ]  `updated_at` 이 갱신된다
- [ ]  content 비워서 저장 시 버튼이 비활성화된다
